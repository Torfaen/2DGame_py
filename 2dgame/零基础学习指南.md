## 📚 第一步：理解游戏是怎么"动"起来的

### 游戏就是"画动画"
想象你小时候看的动画片：
- 一张张画快速播放 = 动画
- **游戏也是这样！** 电脑每秒画60张图（60帧），快速播放就变成"会动"的游戏

### 游戏的基本流程（永远重复这三步）
```
1. 检查：玩家按了什么键？
2. 计算：人物应该往哪走？
3. 画画：在屏幕上画出新位置
```

## 🗂️ 第二步：看懂项目文件结构

把游戏想象成一个**玩具工厂**：

```
📁 2dgame/                    ← 整个游戏工厂
│
├── 📁 src/                   ← 游戏的大脑（所有逻辑代码）

│   ├── game_manager.py       ← 🎮 游戏的总指挥
│   ├── player.py             ← 👤 玩家的代码
│   ├── bomb.py               ← 💣 炸弹的代码
│   └── map.py                ← 🗺️ 地图的代码
│
├── 📁 assets/                 ← 游戏的"素材库"
│   ├── sprites/              ← 所有图片（角色、炸弹、地图）
│   └── audio/                ← 所有声音（背景音乐、爆炸声）
│
├── 📁 config/                ← 游戏的"设置文件"
│   └── config.yaml           ← ⚙️ 可以改这里调整游戏参数
│
└── 📁 map/                   ← 地图数据（用数字表示地图）
    └── *.json                ← 地图文件
```

## 🎯 第三步：从哪里开始看？（按顺序）

### ⭐ 新手推荐路线

#### 1️⃣ **入口文件** - `src/main2.py` （最简单！）
```
这里只有5行代码，告诉你游戏是怎么启动的：
1. 加载设置 → 2. 创建游戏 → 3. 初始化 → 4. 运行
```

**看这里理解什么？**
- 游戏从哪开始执行
- 整体流程是什么样的

---

#### 2️⃣ **设置文件** - `config/config.yaml` （能看懂！）
```
这是用普通文字写的设置文件，不是代码！

比如：
  speed: 3          ← 玩家移动速度是3
  bomb_power: 7     ← 炸弹爆炸范围是7格
  width: 800        ← 窗口宽度800像素
```

**看这里理解什么？**
- 可以改什么参数
- 怎么调整游戏难度
- 玩家属性是什么

---

#### 3️⃣ **玩家类** - `src/player.py` （核心之一）
```
这个文件控制：
- 玩家怎么移动
- 玩家怎么放炸弹
- 玩家什么时候死
```

**重点看这些函数：**
- `handle_input()` - 检查按键
- `move()` - 移动逻辑
- `place_bomb()` - 放炸弹

**看这里理解什么？**
- 玩家输入怎么变成动作
- 移动是怎么实现的
- 碰撞检测是什么

---

#### 4️⃣ **炸弹类** - `src/bomb.py`
```
这个文件控制：
- 炸弹什么时候爆炸
- 炸弹爆炸范围多大
- 炸弹的动画效果
```

**看这里理解什么？**
- 倒计时怎么实现
- 爆炸范围怎么计算

---

#### 5️⃣ **游戏管理器** - `src/game_manager.py` （最复杂）
```
这是游戏的"大脑"，控制：
- 什么时候更新玩家
- 什么时候检查爆炸
- 什么时候判断胜负
```

**看这里理解什么？**
- 游戏主循环是什么
- 各个系统怎么协调工作

## 🔍 第四步：理解关键概念（用大白话）

### 1. **类（Class）** = 图纸
```
就像做饼干的模具：
- Player类 = 做玩家的模具
- 每个玩家都是这个模具做出来的
- 但每个玩家的位置、血量可以不同
```

### 2. **函数（Function）** = 动作
```
就像遥控器的按钮：
- move() = 按了"移动"按钮
- place_bomb() = 按了"放炸弹"按钮
```

### 3. **变量（Variable）** = 盒子
```
就像储物箱：
- self.x = 存储玩家的X坐标
- self.alive = 存储玩家是否还活着（True/False）
```

### 4. **列表/数组** = 装多个东西的盒子
```
bombs_group = [炸弹1, 炸弹2, 炸弹3]
就像一排座位，可以放很多炸弹
```

## 🛠️ 第六步：动手实践（改简单的东西）

### 最简单的改动

#### 1. 改玩家速度
**文件：** `config/config.yaml`
```yaml
players:
  - id: 0
    speed: 3    ← 改成 5，玩家就跑更快了
```

#### 2. 改炸弹数量
**文件：** `config/config.yaml`
```yaml
players:
  - id: 0
    bomb_count: 2    ← 改成 5，就能放更多炸弹
```

#### 3. 改窗口大小
**文件：** `config/config.yaml`
```yaml
windows:
  window:
    width: 800    ← 改成 1000，窗口就变宽了
    height: 600   ← 改成 800，窗口就变高了
```

## 🎓 第七步：看懂代码的技巧

### 看代码时问自己：

1. **这是什么？**
   - 变量名通常说明它的用途
   - `player.speed` = 玩家的速度

2. **它做什么？**
   - 函数名通常说明它的功能
   - `move()` = 移动
   - `place_bomb()` = 放置炸弹

3. **什么时候执行？**
   - 在游戏循环里 = 每秒执行60次
   - 在 `update()` 里 = 每帧更新
   - 在事件里 = 按了某个键才执行

4. **如果看不懂某行？**
   - 跳过！先看整体结构
   - 看注释（`#` 后面的文字）
   - 看变量名和函数名猜意思

## ❓ 常见问题

**Q: 完全看不懂代码怎么办？**
A: 正常！先不用看懂每行代码，先理解"这段代码在做什么"，就像看电影先理解剧情，再研究拍摄技巧。

**Q: 我应该先学Python吗？**
A: 不必须！可以先通过这个项目学，遇到不懂的再查。但学基础Python会有帮助。

**Q: 为什么代码这么复杂？**
A: 因为游戏要考虑很多东西：碰撞、动画、输入、状态管理... 但是分开看，每个部分都不难。

**Q: 我应该从哪里开始改？**
A: 从 `config.yaml` 开始！这里不需要写代码，只需要改数字，很容易看到效果。
